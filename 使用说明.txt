【您只需要记住 3 个操作】

───────────────────────────────────────────────────
一、日常更新流程（最常用）
───────────────────────────────────────────────────

  第 1 步：改内容
    · 在 Cursor 对话里说明需求，我来改 OAO.html
    · 不要自己乱改代码文件

  第 2 步：本地测试
    · 双击：打开OAO.bat
    · 浏览器会打开 http://127.0.0.1:8777/OAO.html
    · 测试完关闭「OAO Preview Server」黑色窗口

  第 3 步：上传发布
    · 双击：一键上传GitHub.bat
    · 看到「上传成功」即可
    · 约 2 分钟后打开正式网站查看

───────────────────────────────────────────────────
三、第一次上传 / Token 过期
───────────────────────────────────────────────────

  1. 浏览器打开（需登录 GitHub 账号 wh9007）：
     https://github.com/settings/tokens/new?scopes=repo

  2. 点 Generate token，复制 ghp_ 开头的文字

  3. 双击「一键上传GitHub.bat」
     · 弹出记事本 → 粘贴 Token → 保存 → 关闭
     · 回到黑色窗口看结果

  ※ Token 是密码，不要发给任何人


───────────────────────────────────────────────────
四、外网 AI 功能（可选）
───────────────────────────────────────────────────

  访客在 GitHub 网站用 AI 时，需要您电脑在线并运行：

    1. 双击「启动Tunnel.bat」（窗口不要关）
    2. Ollama 已运行（端口 11434）
    3. AnythingLLM 已运行（端口 3001）

  本地预览（打开OAO.bat）时，一般不用开 Tunnel。


───────────────────────────────────────────────────
五、常见问题
───────────────────────────────────────────────────

  Q：上传失败？
  A：重新生成 GitHub Token，再双击「一键上传GitHub.bat」。
     详细错误见 upload-log.txt，截图发给 Cursor 助手。

  Q：网站打不开或跳到 oao.eth？
  A：浏览器缓存了旧跳转。按 Ctrl+Shift+Delete 清除缓存，
     或用无痕窗口打开正式网站地址。

  Q：GitHub Pages 要不要填 oao.eth？
  A：不要！Custom domain 必须留空。
     设置页：https://github.com/wh9007/oao-platform/settings/pages

  Q：外网 AI 不工作？
  A：确认「启动Tunnel.bat」在运行，且 Ollama、AnythingLLM 已启动。
